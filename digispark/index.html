What is Digispark?

In order to know Digispark, and better undertand how it works, we must first learn a little bit about the ruby micro framework for robotics used to interact with the Digispark platform.

What is artoo?

Artoo is a micro-framework for robotics. It provides a simple, but powerful Domain-Specific Language (DSL) for robots and physical computing.

With Artoo, you can easily connect multiple, different hardware devices together at the same time into a single solution. Artoo makes telling robots what to do incredibly simple and fun.

Ruby versions supported: Ruby 2.0, Ruby 1.9.3, JRuby 1.7.4, and Rubinius 2.0 Artoo is conceptualy influenced by Sinatra ( Sinatra ) as well as borrowing some code from it. Artoo provides a robust actor-based messaging architecture, that can support fully multi-threaded operation and high-concurrency, as long as it is supported by the Ruby version in which it is executing.

This means you will need to use JRuby or Rubinius for maximum concurrency. To a large extent, this is due to being built on top of Celluloid ( Celluloid ), Celluloid::IO ( Celluloid::IO ), and Reel ( Reel ).

The Digispark microcontroller

The Digispark is an Attiny85 based microcontroller development board similar to the Arduino line, only cheaper, smaller, and a bit less powerful. With a whole host of shields to extend its functionality and the ability to use the familiar Arduino IDE the Digispark is a great way to jump into electronics, or perfect for when an Arduino is too big or too much.

Installing Littlewire communication protocol in the Digispark

First thing to do is to setup your Digispark microcontroller for communication with Artoo. Artoo communicates with the Digispark using the Littlewire communication protocol ( http://littlewire.cc/ ), so we have to make sure this protocol has been installed to the Digispark firmware, if this is the case, you can start working with Artoo right away.

Nevertheless if the Digispark does not have the littlewire protocol installed yet, you can install it easily, by first installing the artoo-digispark gem, and then running the command 'artoo install digispark'. The program will prompt you to plug in the Digispark, and then once you do, it will install the firmware automatically.

Connecting digispark to your computer

Now that we have the littlewire communication protocol installed in the Digispark we can use it with Artoo. When you plug it, you should be able to use it with Artoo.

Installing artoo and artoo-digispark gems

Installation is pretty simple, just install the artoo-digispark gem and you are set. 

gem install artoo-digispark

Writing your robot code:

Now you are ready to write your own code. Take a look at the examples directory in the artoo-digispark github repository (or clone the repository to your computer), for a whole bunch of code you can use to help get started. We recommend using Test-Driven Robotics (TDR) with your preferred test frameworks.

Running your robot:

ruby ./examples/blink.rb

What the code looks like

Example: Digispark with LED and a button, using the Littlewire protocol.

require 'artoo'

# Uses Digispark USB board (http://digistump.com/products/1) 
# with Little Wire protocol (http://littlewire.cc)

connection :digispark, :adaptor => :littlewire, :vendor => 0x1781, :product => 0x0c9f
device :board, :driver => :device_info
device :led, :driver => :led, :pin => 1

work do
  puts "Firmware name: #{board.firmware_name}"
  puts "Firmware version: #{board.version}"

  every 1.second do
    led.toggle
  end
end
